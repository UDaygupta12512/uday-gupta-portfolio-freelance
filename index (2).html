🏗 How This Site Was Built

Philosophy

Built as a single HTML file — no framework, no bundler, no build step. Every animation, modal, API integration, and theme toggle runs in ~340KB of vanilla HTML/CSS/JS. The constraint was intentional: if you understand the stack deeply enough, you don't need a framework to build something polished.

Animation Stack

All scroll-driven animations use GSAP 3.12 + ScrollTrigger — registered via CDN, no install. Scroll animations use gsap.from() with scrollTrigger: {start:'top 95%'} on each element independently, staggered via stagger: 0.08. The hero canvas is a hand-coded particle field using the Web Canvas API — no library.

GSAP 3.12 ScrollTrigger Canvas API CSS Keyframes IntersectionObserver

Theming System

Dark/light mode uses a single [data-theme="light"] attribute on <html>. Every color in the site is a CSS custom property (var(--bg), var(--text), etc.) redefined under that selector. The theme attribute is set in a tiny inline script in <head> to prevent flash-of-wrong-theme on reload — before the first paint happens.

Integrations

Contact form uses Formspree as primary (lead dashboard) → EmailJS as fallback → mailto: as last resort. GitHub stats fetch lazily via IntersectionObserver (only when the section enters viewport). Blog content is curated static data with a JSON structure that can be hot-swapped for a live Dev.to API fetch. The "now" status panel reads from /status.json — a pattern borrowed from Derek Sivers' nownownow.com.

Mobile nav is a full-screen overlay that renders the same links as the desktop nav, triggered by a hamburger that animates 3 lines → X.
